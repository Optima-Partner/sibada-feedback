## 1. 系统概览（非技术也能看懂）

### 1.1 平台由哪些部分组成

平台是一组协作的服务，跑在本院的服务器上。你不需要记住细节，大致知道"谁负责什么"即可：

| 组成部分 | 通俗职责 |
|----------|----------|
| **前端（网站）** | 运营打开的工作台页面 |
| **对话网关 + AI 运行时** | 处理对话、驱动 AI、调用各项能力 |
| **业务后端** | 脚本、数字人、音频、视频等业务逻辑 |
| **知识库** | 存放专业资料（企业库）和个人偏好库，供 AI 引用 |
| **生成服务** | 图片 / 海报 / 语音等生成能力（对接 GrsAI、DashScope 等） |
| **内容调研（Scout）** | 抖音 / 小红书 / 视频号数据分析（对接 TikHub） |
| **登录服务（Logto）** | 账号注册、登录、审核门禁 |
| **数据库 / 缓存 / 存储** | 存业务数据、会话缓存、视频音频文件 |

这些服务一起构成了运营看到的"一个平台"。

### 1.2 部署在哪

- **阿里云 ECS（服务器）**：跑上面所有服务。
- **阿里云 RDS（数据库）**：存业务与登录数据。
- **阿里云 NAS（文件存储）**：存知识库、会话工作空间。
- **阿里云 ACR（镜像仓库）**：存放各服务的程序镜像，升级时用。

以上资源都在**本院的阿里云账户**下，由 Optima 协助运维。

### 1.3 域名与访问入口

平台主域名：**`ops.spider-net.com.cn`**（示例，以实际为准）。运营日常只用**主入口网址**即可；其余子域（登录、接口、网关等）由系统内部使用，无需关心。

## 2. 🟢 日常可自助

本章分两类：**A. 不需要服务器权限**（任何管理员都能做）、**B. 需要服务器访问**（仅当本院有技术人员持有服务器登录权限时）。没有服务器权限时，B 类一律走 [🔴 报障](#38-怎么报障)。

### A. 不需要服务器权限

#### 2.1 账号与区域管理（在平台"系统设置"里做）

用**管理员账号**登录平台，进入**系统设置**，可以：

- **审核放行新用户**：新人注册后默认不能用，需你在这里**批准**。
- **禁用 / 启用**账号：暂停或恢复某人的使用。
- **备注**账号：给账号加说明（如"XX 科室运营"）。
- **删除**账号（**不可逆**）。
- **区域管理**：创建 / 编辑 / 删除区域，并把用户和医生分配到区域。

> ⚠️ **注意**
> - **删除 / 禁用是对人的操作，动手前确认清楚是哪一个账号**；**拿不准时优先用"禁用"（可随时恢复）而不是"删除"（不可逆）**。
> - **把某个运营提升为管理员**不是这里的按钮 —— 需要在登录服务（Logto）侧配置、可能要重新部署，属 🔴 [请联系 Optima](#3--请联系-optima)。
> - 拉新运营进来的标准流程：**本人注册 → 你在系统设置里批准**。

#### 2.2 第三方账户充值与密钥（最常见的"故障"其实是"没钱"）

平台的很多能力对接了第三方服务。**这些服务的账户由本院持有**，用完额度就会以"功能故障"的形式表现出来。**先看是不是没钱，再当故障报**。

| 服务 | 平台里负责什么 | "没钱 / 出问题"的典型症状 | 怎么处理 |
|------|----------------|---------------------------|----------|
| **HeyGen** | 数字人训练、视频渲染 | 出片报"没积分 / 渲染失败" | 登录 HeyGen 后台**购买 API credits**（见下方坑① ） |
| **SiliconFlow（硅基流动）** | AI 对话回复 | AI 不回复 / 回复报错 | 登录后台充值。注意用**国内站**账户 |
| **GrsAI** | 图片 / 海报生成 | 生成图片报"生成失败" | 登录后台充值 / **检查密钥限额**（见下方坑②） |
| **DashScope（阿里云百炼）** | 语音输入 / 转写 | 语音功能报错 | 阿里云百炼后台充值 |
| **TikHub** | 抖音 / 小红书 / 视频号分析 | 调研失败 / 返回为空 | 登录后台充值 |

> ⚠️ **两个真实踩过的坑（务必记住）**
> - **坑① HeyGen 有两个钱包**：网页端看到的"套餐积分"和**API credits** 是**两个池子**。数字人出片用的是 **API credits**。网页积分再多，API credits 为 0 一样报"没积分" —— 要专门买 **API credits**。
> - **坑② GrsAI 的"积分"不等于"密钥可用额度"**：账户里显示有很多积分、密钥也没错，仍可能报"额度不足" —— 因为**给这把密钥单独设了消费限额**。到 GrsAI 后台**确认该密钥没有被限额、且在已充值的账户下**。（本平台已实测过此坑。）
>
> ⚠️ **不要自行更换第三方密钥**：更换密钥（尤其 HeyGen）会导致已训练的数字人 / 音色失效，且需要改服务器配置。**充值在各服务后台做即可，不涉及换密钥**；确需换密钥请走 🔴 联系 Optima。

#### 2.3 看平台是否"活着"

最简单的判断：**用浏览器打开平台主入口网址**。

- 能正常打开、能登录、能发消息 → 平台在线。
- 打不开 / 登录异常 / 发消息长时间无响应 → 见 [第 4 章故障速查](#4-故障速查表)；多数属 🔴，请报障。

### B. 需要服务器访问（仅本院技术人员）

> 以下操作需要**服务器（ECS）登录权限**。若本院无人持有该权限，**整节归入 🔴 报障**，不要尝试自行获取权限操作。

#### 2.4 看日志（只读，帮助定位问题）

在服务器上查看某个服务的近期日志，用于报障时附带信息。**只看不改**：

```bash
# 查看某服务最近日志（把 <服务> 换成实际容器名，如 sibada-generation / sibada-backend / sibada-web / sibada-gw-core）
sudo docker logs --tail 100 <服务>
```

看到的报错原文，**原样复制**给 Optima，能大幅加快排查。

#### 2.5 重启单个服务（仅限下表可安全重启的服务，强护栏）

某个功能卡住、且确认不是"没钱"时，可**重启对应的单个服务**试试。这是相对安全的操作，但**只限下表这几个**：

| 可以重启 | 对应功能 | 影响 |
|----------|----------|------|
| `sibada-web` | 前端网站 | 几乎无感，重启几秒 |
| `sibada-generation` | 图片 / 语音生成 | 只会打断当时那一次生成（重试即可），不影响对话历史 |
| `sibada-scout` | 抖音 / 小红书分析 | 不影响进行中的对话 |
| `sibada-gw-core` | 对话网关 | ⚠️ **会断开当前在线的对话连接**（用户刷新即可重连，**不丢已保存的数据**），请在没人使用时做 |

```bash
sudo docker restart <上表中的服务名>
```

> 🔴 **绝对不要**自行重启 / 停止下面这些（含状态、或牵一发动全身，请走 🔴 报障）：
> - **`sibada-backend`（业务后端）** —— 含进行中的**出片 / 音频 / 数字人训练轮询**任务，重启会打断这些任务
> - **`sibada-agent`（AI 运行时）**
> - **数据库、登录服务（logto）、缓存（redis）、对象存储（minio）、反向代理（caddy）**
>
> ⚠️ 重启一次不好使，**不要连着重启或扩大范围** —— 说明不是简单卡顿，请报障。

#### 2.6 看磁盘空间（预防"会话变空"）

服务器**根磁盘满**会导致"发消息无响应、会话刷新后空白"（曾发生过）。技术人员可定期看一眼：

```bash
df -h /        # 根盘；使用率长期 > 85% 需警惕
df -h /mnt/data
```

> 🔴 **磁盘接近满 / 扩容 / 清理属于 Optima 的事**，看到快满了**报障**即可，不要自行删文件（易误删运行中数据）。

### 2.7 日常自检清单（建议每周过一遍）

- [ ] 打开平台主入口，能登录、能发消息、能出片。
- [ ] 五个第三方账户余额是否充足：**HeyGen（API credits）/ SiliconFlow / GrsAI / DashScope / TikHub**。建议各自开启余额告警。
- [ ]（有服务器权限时）`df -h /` 根盘使用率是否正常（< 85%）。
- [ ] 有无用户反馈的异常，需要报障的已记录。

## 3. 🔴 请联系 Optima

以下操作**请不要自行进行**，联系 Optima 处理。列在这里是为了让你知道"这类需求归我们"。

| 需求 | 说明 |
|------|------|
| **3.1 部署 / 版本更新** | 上线新功能、更新平台版本（需重建镜像、切流量） |
| **3.2 数据库迁移 / 数据变更** | 表结构变更、批量数据处理、备份恢复 |
| **3.3 磁盘扩容 / 存储迁移** | 磁盘接近满、扩容、数据迁移 |
| **3.4 回滚 / 故障恢复** | 版本回退、服务大面积异常、数据异常 |
| **3.5 证书 / 域名 / 安全组** | HTTPS 证书、域名解析、网络端口放行 |
| **3.6 更换第三方密钥** | 见 [2.2](#22-第三方账户充值与密钥最常见的故障其实是没钱) |
| **3.7 授予 / 变更管理员角色** | 把运营提升为管理员需在登录服务（Logto）侧配置、可能重新部署 |

> 注：**企业医学知识库的日常增删改由运营自己在对话里完成**（改动即时共享全员），不需要联系 Optima —— 见《使用手册》§2.1。

### 3.8 怎么报障

联系 Optima 时，**尽量带齐这些信息**（能把排查时间从小时级压到分钟级）：

1. **发生时间**（大概几点）。
2. **谁遇到的**（账号 / 手机号）。
3. **在做什么操作**（例如"用医生 A 出片时"）。
4. **报错原文或截图**（最重要 —— 原样复制，不要转述）。
5. （有服务器权限时）相关服务的日志片段（见 [2.4](#24-看日志只读帮助定位问题)）。
6. **影响范围**：只有你一个人 / 多人 / 全站打不开。

> 📌 报障渠道：以与 Optima 约定的对接方式为准（微信 / 邮件 / 工单）。全站不可用等紧急情况，请走约定的紧急联系人。

## 4. 故障速查表

| 现象 | 很可能的原因 | 归属 | 处理 |
|------|--------------|------|------|
| 出片报"没积分 / 渲染失败" | HeyGen **API credits** 用尽 | 🟢 | 后台买 API credits（[2.2](#22-第三方账户充值与密钥最常见的故障其实是没钱)） |
| 生成图片 / 海报报"生成失败" | GrsAI 额度不足**或密钥被设了消费限额** | 🟢 | 后台充值 / 解除密钥限额（[2.2](#22-第三方账户充值与密钥最常见的故障其实是没钱) 坑②） |
| AI 不回复 / 回复报错 | SiliconFlow 额度 | 🟢 | 后台充值 |
| 语音输入 / 转写不工作 | DashScope 额度 | 🟢 | 百炼后台充值 |
| 抖音 / 小红书分析失败或为空 | TikHub 额度 / 网络波动 | 🟢 | 充值 / 稍后重试 |
| 某个功能卡住（非"没钱"） | 单个服务异常 | 🟢(有权限) | 重启对应可安全重启的服务（[2.5](#25-重启单个服务仅限下表可安全重启的服务强护栏)） |
| **发消息无响应 / 会话刷新后空白** | 服务器**根盘满**等系统问题 | 🔴 | 报障（可能需扩容 / 清理） |
| 站点打不开 / 登录异常 | 证书 / 网关 / 系统问题 | 🔴 | 报障 |
| 数据异常 / 疑似丢失 | 数据层问题 | 🔴 | **立即报障，不要自行改动** |
